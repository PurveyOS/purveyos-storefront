THE FIX IN 30 SECONDS
================================================================================

BEFORE:
  Anonymous request → useStorefrontData queries products with supabase client
  → RLS evaluates 5 overlapping policies
  → One policy has expensive subquery: (tenant_id IN (SELECT profiles...))
  → Subquery scans profiles table
  → Timeout (57014) ❌

AFTER:
  Anonymous request → useStorefrontData creates tenantClient with x-tenant-id header
  → Query includes header: GET /products ... x-tenant-id: uuid
  → RLS evaluates 1 simple policy per role
  → Policy checks: is_online=true AND tenant_id=header (no subqueries)
  → Regex validates header format
  → Fast response (<300ms) ✅
